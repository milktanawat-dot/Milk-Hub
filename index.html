💰 แผนการลงทุน/สมทบเงิน

      

      

        
        
      

      
เหมาะกับสินทรัพย์ที่หักเงินเดือนอัตโนมัติทุกเดือน (เช่น PVD) เลือก "รายเดือน" — เหมาะกับเงินฝาก/สมทบสหกรณ์ที่ใส่เป็นก้อนปีละครั้ง เลือก "รายปี"

      

        
        
        
        
      

      
ผลตอบแทน/ปันผลคาดหวัง/ปี = ค่าเริ่มต้นในการคำนวณ ไปแก้เป็นตัวเลขปันผลจริงที่ประกาศแต่ละปีได้ที่การ์ดเป้าหมาย (▼ ตั้งค่ารายปี) — เพดานมูลค่าสูงสุดใช้กับกรณีมีเพดานหุ้น เช่น สหกรณ์ออมทรัพย์ ระบบจะหยุดสมทบเพิ่ม "และ" หยุดนับผลตอบแทน/ปันผลเพิ่ม เมื่อมูลค่ารวมถึงเพดานนี้แล้ว (มูลค่าจะไม่มีวันเกินเพดานที่ตั้งไว้)


      

      

        
💼 คำนวณเงินลงทุน/เดือนจากเงินเดือน PVD / กองทุนสำรองเลี้ยงชีพ

        
      

      

        
        
        
        
        
      

      
กรอกเงินเดือนและ % แล้วกด "ใช้เป็นเงินลงทุนเพิ่ม/เดือน" — ดึงเงินเดือนตั้งต้นจากสลิปล่าสุดในแท็บภาษีให้อัตโนมัติถ้ามี

      

        
      

      
เมื่อกด "ใช้เป็นเงินลงทุนเพิ่ม/เดือน" ระบบจะผูกตารางปรับเงินสมทบรายปีในการ์ดเป้าหมายให้อัตโนมัติเป็น % เงินเดือน × เงินเดือนที่คาดว่าจะขึ้นในแต่ละปี — ไปที่การ์ดเป้าหมายแล้วกด "▼ ตั้งค่ารายปี" เพื่อดูตัวเลขคาดการณ์ หรือแก้เป็นตัวเลขจริงที่ได้รับในปีนั้นๆ ได้เลย

      


      

        
        
      

      
เลือก "พอร์ตรวมทั้งหมด" หรือ "Net Worth รวม" เพื่อตั้งเป้าหมายภาพรวมแบบเดียวกับ Dashboard ทรัพย์สิน หรือเลือกสินทรัพย์รายตัว (กองทุน/หุ้น/คริปโต) เพื่อตั้งเป้าหมายเฉพาะรายการนั้น — ระบบคำนวณอายุจากวันเกิด หาปีที่เหลือถึงเกษียณ แล้วทบต้นเงินลงทุน/เดือนรายเดือนจากมูลค่าปัจจุบันจริง (วันเกิดกรอกครั้งเดียว ใช้ร่วมกันทุกเป้าหมาย)

โหมดนี้ "กู้คืน" ต้องทำเอง: เปิด Google Drive → โฟลเดอร์ Milk Hub Backup → เปิดไฟล์ล่าสุด → ดาวน์โหลด → มา Import ในการ์ด "สำรอง/ย้ายข้อมูล" ด้านบน


        

          ✅ วิธีตั้งค่าด้วย Google Apps Script (แนะนำ — ~5 นาที ไม่ต้องมีตัวกลาง ไม่ติดปัญหา org)
          
    
            
  1. ไปที่ script.google.com → ล็อกอินด้วย gmail เดิม (milktanawat@gmail.com) → กด New project
    2. 
            
  2. ลบโค้ดเดิมในกล่องทั้งหมดออก แล้ววางโค้ดนี้แทน:
    3. 
          

          
function doPost(e) {
  var folderName = "Milk Hub Backup";
  var fileName = "milk-hub-backup.json";
  var folders = DriveApp.getFoldersByName(folderName);
  var folder = folders.hasNext() ? folders.next() : DriveApp.createFolder(folderName);
  var old = folder.getFilesByName(fileName);
  while (old.hasNext()) { old.next().setTrashed(true); }
  folder.createFile(fileName, e.postData.contents, MimeType.PLAIN_TEXT);
  return ContentService.createTextOutput("OK");
}

          
    
            
  1. ตั้งชื่อโปรเจกต์มุมซ้ายบน เช่น Milk Hub Backup (ไอคอนดินสอ) → กด Save (รูปแผ่นดิสก์)
    2. 
            
  2. กด Deploy → New deployment → กดไอคอนเฟือง ⚙️ เลือกประเภทเป็น Web app
    3. 
            
  3. ตั้งค่า: Execute as = Me · Who has access = Anyone → กด Deploy
    4. 
            
  4. จะมีหน้าต่างขอสิทธิ์ (Authorize access) → เลือกบัญชี gmail → ถ้าเจอ "Google hasn't verified this app" ให้กด Advanced → Go to Milk Hub Backup (unsafe) → Allow (ปลอดภัย เพราะเราเป็นคนเขียนโค้ดเอง)
    5. 
            
  5. คัดลอก Web app URL ที่ได้ (ขึ้นต้นด้วย https://script.google.com/macros/s/…/exec) → มาวางในช่องด้านบน → กด "บันทึก URL"
    6. 
          

          ทุกครั้งที่กด "สำรองตอนนี้" หรือ auto-backup แอปจะส่งข้อมูลตรงไปที่สคริปต์นี้ แล้วมันจะเซฟไฟล์ลง Google Drive ของเราเองทันที — ไม่ผ่านตัวกลางใคร ไม่มีปัญหาบัญชีองค์กรมาบล็อกแน่นอน
        


        

          ทางเลือกอื่น: ผ่าน Make.com + OneDrive (ถ้าไม่ติดปัญหา org)
          
    
            
  1. ไปที่ make.com → Sign up ฟรี
    2. 
            
  2. สร้าง scenario → โมดูลแรก Webhooks → Custom webhook → คัดลอก URL มาวางช่องด้านบน
    3. 
            
  3. โมดูลที่ 2: OneDrive → Upload a file → เชื่อมบัญชี Microsoft ส่วนตัว
    4. 
            
  4. Data ใส่ {{1.body}} (เนื่องจากตอนนี้แอปส่งแบบ text/plain แล้ว ควรเจอ field "body" ตรงๆ ไม่ต้องใช้ toJSON)
    5. 
          

          ⚠️ ถ้าเน็ตเวิร์กที่ใช้บล็อกบัญชี Microsoft ส่วนตัว (เจอ error "can't sign in with personal account") ให้ใช้ Google Apps Script ด้านบนแทนจะชัวร์กว่ามาก
        

      


      

        
        
        

          
          
          
          
        

        
        

        

          วิธีตั้งค่า (ต้องมี Azure tenant เป็นของตัวเอง)
          
    
            
  1. ไปที่ portal.azure.com → ค้นหา "App registrations" → New registration
    2. 
            
  2. ตั้งชื่อ เช่น Milk Hub · Supported account types เลือก Accounts in this organizational directory only
    3. 
            
  3. Redirect URI เลือก platform Single-page application (SPA) แล้วใส่ URL ของเว็บนี้ เช่น https://ชื่อ.github.io/milk-hub/
    4. 
            
  4. กด Register แล้วคัดลอก Application (client) ID มาวางในช่องด้านบน → กดเชื่อมต่อ
    5. 
          

          ⚠️ ต้องมี directory/tenant ก่อนถึงจะสมัคร App ได้ — ถ้าเจอ error "does not exist in tenant" หรือสมัคร M365 Dev Program ไม่ผ่าน ให้ใช้แท็บ "ผ่าน Make.com" แทนจะง่ายกว่ามาก
        

      

    

    

      
📱 ใช้บนมือถือเหมือนแอป

      

        เปิดไฟล์นี้ในเบราว์เซอร์มือถือ (Safari/Chrome) → กด Share → "Add to Home Screen" จะได้ไอคอนหน้าจอโฮม เปิดแล้วเต็มจอไม่มีแถบ URL
        · ⚠️ ข้อมูลผูกกับเบราว์เซอร์/เครื่องที่เปิดไฟล์เท่านั้น เปิดไฟล์เดียวกันจากเครื่องอื่นจะเห็นข้อมูลคนละชุด — ใช้ "ดาวน์โหลด Backup" ด้านบนเพื่อย้ายข้อมูลข้ามเครื่อง

ℹ️ ระบบข้อมูล

      

        ข้อมูลทั้งหมดเก็บใน localStorage ของเบราว์เซอร์ที่เปิดไฟล์นี้ (ไม่ผ่านอินเทอร์เน็ต ไม่ส่งไปที่ไหน) — เปิดไฟล์เดียวกันจากเบราว์เซอร์/เครื่องอื่นจะไม่เห็นข้อมูลชุดนี้ ต้อง Export/Import backup เพื่อย้ายข้อมูล
        · ล้างประวัติเบราว์เซอร์ (clear browsing data) อาจลบข้อมูลนี้ไปด้วย ควร Export backup เก็บไว้เป็นระยะ
